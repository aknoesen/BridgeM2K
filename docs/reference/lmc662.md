# LMC662 — behavioural op-amp model (reference)

The TI **LMC662** dual CMOS op-amp is the op-amp used in the EEC1 course. The twin models it
behaviourally so a drawn circuit shows the same first-order non-idealities a student sees on the
bench. Source: TI LMC662 datasheet (SNOSBX6), https://www.ti.com/lit/ds/symlink/lmc662.pdf.

## Datasheet figures used

| Parameter | Value | In the model? |
|-----------|-------|---------------|
| Open-loop voltage gain (Aol) | 126 dB ≈ 1.995×10⁶ V/V | yes |
| Gain·bandwidth product (GBW) | 1.4 MHz | yes (dominant pole at GBW/Aol ≈ 0.7 Hz) |
| Output swing | rail-to-rail | yes (clip to supply rails) |
| Slew rate | 1.1 V/µs | not yet (kept as a constant for reference) |
| Phase margin | 50° | implied by the single pole; not separately modelled |
| Supply range | 4.75–15.5 V, single-supply capable | clip rails default ±5 V (M2K supplies) |
| Input bias current | 2 fA (CMOS) | not modelled (treated ideal) |
| Input offset voltage | 3 mV | not modelled |
| ISS | 400 µA/amplifier | not modelled |
| Channels | dual | n/a (one model per placed op-amp) |

## How it is built (`core/netlist.ts`, `opampLines`)

Single-pole macromodel + output clip:

```
E<id>  xop<id>_g 0  inP inN  1.995e6           ; open-loop gain Aol
Rp<id> xop<id>_g xop<id>_o  1k                 ; dominant pole R …
Cp<id> xop<id>_o 0  Cp                          ; … with Cp = 1/(2π·fp·Rp), fp = GBW/Aol ≈ 0.7 Hz
B<id>  out 0  V = max(vneg, min(vpos, V(xop<id>_o)))   ; rail-to-rail output clip
```

- **AC (.ac):** the RC pole gives a −20 dB/decade rolloff; closed-loop bandwidth = GBW / closed-loop
  gain (e.g. a ×10 non-inverting amp rolls off at ≈140 kHz). The clip linearises away in small-signal
  AC, so the Bode plot is the bandwidth story.
- **Transient (.op/.tran):** the B-source clamps the output to the supply rails, so an over-driven
  stage clips at ±5 V instead of reaching the ideal value.

Verified in `core/spice.test.ts` ("LMC662 behavioural op-amp"): ×10 amp bandwidth ≈140 kHz; a ×10 of
2 V clips at +5 V where the ideal op-amp reaches 20 V.

## Not modelled yet (future)

- **Slew rate** (1.1 V/µs) — needs a nonlinear rate limiter; visible on fast large-signal edges.
- **Input offset / bias current**, finite CMRR, noise.
- **Rails tracking the Power Supply instrument** — the clip rails are currently the component's
  `supplyPos`/`supplyNeg` (default ±5 V); wiring the op-amp's V+/V− pins to the PSU rails so they
  track automatically is a sensible next step.
